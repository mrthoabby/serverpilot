Create Deploy User

Create a Linux system user for CI/CD deploys, SSH access, or automation. Users are added to the deploy group.

Lowercase letters, numbers, hyphens, underscores. Must start with a letter.
What happens when you click Create:
The server generates a fresh Ed25519 keypair, locks the user's password (SSH-key login only), installs the public key in ~/.ssh/authorized_keys, and shows you the private key once. Copy that private key into your CI provider's secrets (e.g. GitHub Actions SSH_PRIVATE_KEY).
Advanced options
Vault storage encrypts the key at rest with AES-256-GCM (key derived from your dashboard session_secret via HKDF-SHA256). Untick if you want the key to be visible only once, with no recoverable copy on this server.
Paste the public key (ssh-rsa, ssh-ed25519, ecdsa-sha2-*). No password will be set — login only via SSH key.
Password authentication is supported but discouraged. Prefer "Generate keypair" or "Paste public key" for any user that will be used by automation.

Database Connection

The DSN is encrypted at rest with AES-256-GCM. The dashboard never reveals it again once saved — to change the password you delete and re-create the connection.

Postgres: postgres://user:pass@host:port/db?sslmode=require (or libpq key=value form).
MySQL: user:pass@tcp(host:port)/dbname. TLS enabled by default; opt out with ?tls=false.
